To create a transition effect:
1. CSS property:                 height, width, font-size, etc. OR all;
2. duration of the effect:     400ms;
3. timing function:               linear, ease-in, ease-out, etc;
4. delay:                             100ms;

CSS TRANSITIONS
@keyframes ballX {
    0%        { transform: translateX(-275px); }
    100%    { transform: translateX(0px); }
}
.ballPath {
    animation: ballX 2.5s cubic-bezier(0, 0, 0.35, 1);
}
To create an animation:
1. animation-name
2. duration of the effect
3. timing function (linear, ease-in, ease-out, etc)

4. delay
5. iteration-count (1,2,3..., infinite)
6. direction (normal,reverse,alternate, etc)
WHEN TO USE CSS ANIMATION
For smaller, self-contained states for UI elements.
  • navigation menu
  • tooltip
  • slider
  • loading icons
  • for fun

JS ANIMATION: setInterval, setTimeout
CONS:
  • Not intended to be precise
  • Do not take into account what else is happening in the browser
  • Only update the screen when they want to, not when the computer is able to
JS ANIMATION: requestAnimationFrame
function draw() {
    //draw something;
     window.requestAnimationFrame(draw);
}
window.requestAnimationFrame(draw);
Browser will draw the animation at the
next available opportunity
and not at a predetermined interval
WHEN TO USE JS ANIMATION
  • Animate in a truly distinct way on a single element
  • Need to have a significant control over animations or to stop, pause, slow-down or reverse it
  • Animate the scroll position

JS ANIMATION LIBRARIES
  • GSAP - a fast, richly-featured animation platform
  • Velocity.js - a lightweight tool for improving UI animation performance and workflow

PERFORMANCE
Browsers are able to optimize rendering flows. In summary, we should always try to create our animations using CSS transitions/animations where possible. If your animations are really complex, you can may have to rely on JavaScript-based animations instead.
